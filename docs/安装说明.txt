================================================================
                Flow Harvester  -  Windows 安装说明
================================================================

适用 Windows 10 / Windows 11。不需要装 Python，已经打包成 exe。


----------------------------------------------------------------
一、安装前准备（一次性，5 分钟）
----------------------------------------------------------------

需要装 Google Chrome 浏览器：

    https://www.google.com/chrome/

工具会调用本机已安装的 Chrome 来登录 Google 账号，所以这是必需的。
其他依赖（Python 等）一律不需要装。


----------------------------------------------------------------
二、首次启动（1 分钟）
----------------------------------------------------------------

1. 把收到的 FlowHarvester-bundle.zip 解压到一个固定位置，
   建议路径不要太深，不要带空格 / 中文，例如：

       D:\FlowHarvester\

2. 打开解压后的文件夹，双击 FlowHarvester.exe。

3. 第一次启动时 Windows Defender / SmartScreen 可能提示
   "未识别的应用"，点 "更多信息" → "仍要运行"。

4. 一个黑色的 cmd 窗口会打开，里面滚动着启动日志。
   几秒后默认浏览器会自动打开 http://127.0.0.1:8080/，
   显示 Flow Harvester 的总览页面。

5. 看到总览页 = 启动成功。


----------------------------------------------------------------
三、日常使用
----------------------------------------------------------------

  双击 FlowHarvester.exe → cmd 窗口出现 → 浏览器自动打开仪表盘

要点：

* cmd 窗口里滚动的是实时日志（worker / scheduler / web 服务），
  不要关 cmd 窗口。关掉 cmd 等于关掉整个工具。

* 所有操作都在浏览器里完成，不在 cmd 里。

* 如果浏览器没自动打开，看 cmd 窗口顶部的横幅 banner，
  里面写着完整的 Dashboard URL，复制到浏览器即可。

* 要停止：直接关 cmd 窗口，或在 cmd 里按 Ctrl+C。

* 想跟 Windows 一起开机自启动：
  把 FlowHarvester.exe 的快捷方式拖到 shell:startup 文件夹
  （Win+R → 输入 shell:startup → 回车）


----------------------------------------------------------------
四、文件存哪里
----------------------------------------------------------------

  数据 / DB                  %LOCALAPPDATA%\FlowHarvester\flow_harvester.sqlite
  Chrome profile（账号登录态）   %LOCALAPPDATA%\FlowHarvester\profiles\WS_X\
  上传的参考图                %LOCALAPPDATA%\FlowHarvester\assets\<task_id>\
  运行日志                   %LOCALAPPDATA%\FlowHarvester\logs\
  启动崩溃记录                %LOCALAPPDATA%\FlowHarvester\logs\crash.log

  采集到的视频（重要）       %USERPROFILE%\Documents\FlowHarvester\output\

说明：

* %LOCALAPPDATA% 一般是 C:\Users\<你的用户名>\AppData\Local，
  是隐藏文件夹。直接在文件资源管理器地址栏粘贴 %LOCALAPPDATA%
  回车就能进。

* %USERPROFILE%\Documents 就是「文档」文件夹。

* 任务详情页有 "打开文件夹" 按钮，会自动跳转到对应输出目录。


----------------------------------------------------------------
五、卸载 / 更新
----------------------------------------------------------------

更新到新版：
  删旧的解压文件夹，把新版 zip 解压到同位置即可。
  数据 / 账号 / 历史任务全保留（它们都在 %LOCALAPPDATA%\FlowHarvester\
  里，不在 exe 旁边）。

删 DB 重新开始：
  删除 %LOCALAPPDATA%\FlowHarvester\flow_harvester.sqlite，
  下次启动会重建。

彻底卸载：
  1. 删 exe 解压文件夹
  2. 删整个 %LOCALAPPDATA%\FlowHarvester\ 文件夹
  3. 删 %USERPROFILE%\Documents\FlowHarvester\ 文件夹（视频也清掉）


----------------------------------------------------------------
六、常见问题
----------------------------------------------------------------

Q：双击 exe 一闪就消失？
    启动失败。看
        %LOCALAPPDATA%\FlowHarvester\logs\crash.log
    里面有完整的报错信息。cmd 窗口在出错时会暂停等你按回车，
    所以正常情况能直接看到错误信息，截图发给开发者。

Q：Windows Defender SmartScreen 拦截？
    第一次运行会拦截（"未识别的应用"），点 "更多信息" → "仍要运行"。
    后续不会再提示。
    如果是公司管控严的杀毒软件直接删 exe，找 IT 加白名单：
    FlowHarvester.exe 路径 + %LOCALAPPDATA%\FlowHarvester\ 整个文件夹。

Q：浏览器没自动打开？
    看 cmd 窗口顶部的横幅，里面写着
        Dashboard: http://127.0.0.1:<端口>/
    复制这个 URL 到浏览器即可。

Q：8080 端口被占用？
    工具会自动检测端口，被占用就换一个空的。
    cmd 窗口里会打印实际使用的端口号。

Q：Chrome 没装会怎样？
    工具能启动；加账号 → 登录这一步会失败。
    装好 Chrome 即可，不需要重启工具。

Q：怎么知道工具还在跑？
    任务栏有那个黑色 cmd 窗口 = 还在跑。
    关了 cmd 窗口 = 工具完全停止。


----------------------------------------------------------------
七、授权说明
----------------------------------------------------------------

本工具使用时限授权，每个客户拿到的 license.key 文件有 30 天的
有效期（具体期限以开发者签发时为准）。

  * 拿到 zip 包后，里面应该已经包含 license.key 文件，和 exe
    放在同一目录。直接双击 exe 即可启动。

  * cmd 窗口启动时会显示当前授权剩余天数，例如：
        授权: acme-corp, 剩余 23 天 (至 2026-06-04 18:30)

  * 剩余 7 天以内会有⚠提醒，请提前联系开发者续期。

  * 过期后双击 exe 不会启动，会弹出 "授权失败" 提示。
    解决：找开发者要新的 license.key，替换原文件即可，
    数据 / 账号 / 已下载视频全部保留。

  * license.key 也可以放在 %LOCALAPPDATA%\FlowHarvester\
    目录下（这个位置不会被 zip 解压覆盖），更新 bundle 不会丢。


----------------------------------------------------------------
八、给开发者反馈问题
----------------------------------------------------------------

带上以下三样：

  1. 浏览器里出问题页面的截图
  2. %LOCALAPPDATA%\FlowHarvester\logs\app.log 最后 50 行
     （或者整个 logs 文件夹打包发）
  3. 出问题的任务编号（形如 T_20260503T123456_abcdef）

如果是启动失败，附上：

  * %LOCALAPPDATA%\FlowHarvester\logs\crash.log
  * cmd 窗口截图
